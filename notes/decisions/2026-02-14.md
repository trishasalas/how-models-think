# Decisions — 2026-02-14

## Drop Pythia as primary research instrument, pivot to GPT-2

**Context:** Started MLP vs attention decomposition work using Pythia model
family (410m, 1b, 410m-deduped, 1b-deduped). Goal is to understand where
domain-specific accessibility knowledge (e.g. "screen reader") is stored
mechanistically — MLPs vs attention heads — and how that changes with scale.

**Problem:** Pythia has two confounds that make this question unanswerable:

1. **Architecture changes at every scale.** No two adjacent Pythia sizes share
   the same depth AND width. 410m is 24 layers / d_mlp=4096. 1b is 16 layers /
   d_mlp=8192. You can never isolate whether a difference is about MLP capacity
   vs depth vs total parameters. (See full table in src/models.py.)

2. **Parallel attention + MLP.** Pythia uses GPT-NeoX architecture where
   attention and MLP at the same layer compute independently from the same
   input. The MLP can't see what attention did. This makes the decomposition
   question muddier — contributions are independent, not sequential refinement.

**Decision:** Use GPT-2 family instead.

- Sequential attention → MLP (MLP refines what attention found)
- Four sizes with consistent architecture scaling
- Same 4x MLP ratio throughout
- Best-supported model in TransformerLens (most tutorials, most reference code)
- All sizes fit comfortably on 24GB MacBook M5

**What we lose:** Deduplication comparison and 154 training checkpoints per
model. Those were valuable for the earlier emergence work but aren't needed
for the mechanistic question.

**Pythia work preserved:** Earlier research on accessibility concept emergence
across Pythia scales is published and cited. That work stands on its own as
behavioral/emergence analysis. The GPT-2 track is the mechanistic deep dive.

## Restructure repo — shared src/ modules, thin notebooks

**Context:** Had duplicate notebooks across pythia/ and pythia-deduped/
directories. Same logit lens and decomposition code copy-pasted into every
notebook with only the model name changed. Some notebooks loaded the wrong
model due to copy-paste errors.

**Decision:** Extract reusable analysis code into src/ modules:
- `src/models.py` — model loading, metadata, memory cleanup
- `src/logit_lens.py` — logit lens analysis, returns DataFrames
- `src/decompose.py` — attention vs MLP decomposition, returns DataFrames

Notebooks become thin experiment scripts that import from src/. Results
save as CSVs. Comparison happens by loading DataFrames, not squinting at
two browser tabs.

**Deleted:** pythia/ and pythia-deduped/ directories. Results are
reproducible from public models if ever needed again.
