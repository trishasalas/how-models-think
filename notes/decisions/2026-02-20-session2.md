## Run full Pythia scale ladder for paper 3

**Context:** After closing the GPT-2 binding/decomposition argument, ran the
same screen reader prompts across Pythia 160M, 410M, 1B, 2.8B out of curiosity.
Produced unexpectedly rich findings.

**Decision:** Include full Pythia ladder (160M, 410M, 1B, 2.8B) in paper 3.

Useful as lower bound for binding emergence question. Shows strong MLP signal

**1B caveat:** Parallel attention+MLP architecture with expanded MLP (8192)
makes it not directly comparable to other models. Treat as footnote or
limitations item, not as a clean ladder data point.

## Two-prompt design for paper 3

**Context:** Running "A screen reader is" on Pythia 2.8B after the retrieval
prompt produced completely different results — binding heads appear on
definition prompt, MLP dominates on retrieval prompt, same model same concept.

**Decision:** Paper 3 uses two prompt types for each accessibility term:
1. **Retrieval-from-cue:** "A blind person uses a screen" → ` reader`
   Model must retrieve the compound target from partial input.
2. **Definition frame:** "A screen reader is" → ` software`
   Model elaborates on a concept already given as subject.

This is the core finding: binding is prompt-type specific, not concept-specific.
The dissociation between prompt types is the paper 3 argument.

**Why this matters:** Prior binding work used exclusively definition-frame prompts.
The retrieval regime was never tested. Paper 3 closes that gap.
